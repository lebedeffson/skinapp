# MIRA

MIRA — локальное веб-приложение для анализа снимков кожи, ногтей и эмоций.
Нейросети работают на компьютере, а интерфейс можно открыть с телефона в той же
Wi‑Fi сети. Фотографии не отправляются во внешнее облако.

> Результаты моделей не являются медицинским диагнозом и не заменяют врача.

## Быстрый запуск
 
Поддерживаются:

- Windows 10/11 x64;
- Linux x64;
- macOS 14 или новее на Apple Silicon (M1/M2/M3/M4 и новее).

Intel Mac не поддерживается: для него актуальные TensorFlow и PyTorch не
выпускают совместимый набор пакетов.

Нужен 64-битный Python версии 3.12 или 3.13. Рекомендуемая версия — Python 3.12:
[python.org/downloads](https://www.python.org/downloads/).

На всех поддерживаемых системах запускается один и тот же файл `MIRA.py`.
Остальные Python-библиотеки заранее устанавливать не нужно.

### Windows

Откройте PowerShell в каталоге проекта:

```powershell
py MIRA.py
```

Если команда `py` недоступна:

```powershell
python MIRA.py
```

При первом запросе брандмауэра разрешите Python доступ только к частным сетям.

### Linux

```bash
python3 MIRA.py
```

### macOS

```bash
python3 MIRA.py
```

Первый запуск:

1. находит совместимый Python;
2. создаёт локальное окружение `.venv`;
3. устанавливает проверенные CPU-версии библиотек;
4. создаёт локальный HTTPS-сертификат;
5. запускает MIRA и открывает только приложение.

Обычный запуск не создаёт окно или страницу с инструкцией на порту `8500`.
Такая страница включается только явной командой `MIRA.py --certificate`.

Установка библиотек может занять 5–15 минут. Следующие запуски используют
готовое окружение и ничего не скачивают.

## Запуск с телефона

1. Подключите телефон и компьютер к одной Wi‑Fi сети.
2. Запустите `MIRA.py`.
3. Откройте на телефоне напечатанный адрес вида:

   ```text
   https://192.168.1.58:8501
   ```

4. Разрешите браузеру доступ к камере.

Адрес определяется по физическому Wi‑Fi/Ethernet-интерфейсу. VPN, Docker,
виртуальные адаптеры и туннели игнорируются.

### Новый телефон: доверие HTTPS

Камера в мобильном браузере работает только на доверенном HTTPS-сайте.
Для каждого нового телефона один раз выполните:

```bash
python3 MIRA.py --certificate
```

На Windows вместо `python3` используйте `py`. Откроется временная страница
только для скачивания публичного сертификата. После установки нажмите `Ctrl+C`
и запустите MIRA обычной командой.

- Android: Безопасность → Установить сертификат → Сертификат ЦС.
- iPhone/iPad: установите профиль, затем Основные → Об этом устройстве →
  Доверие сертификатам.

Закрытые ключи остаются в `.runtime` на компьютере и никогда не раздаются.

## Управление

Остановить сервер:

```text
Ctrl+C
```

Проверить окружение и модели без запуска:

```bash
python3 MIRA.py --doctor
```

Переустановить библиотеки:

```bash
python3 MIRA.py --reinstall
```

Изменить порт:

```bash
python3 MIRA.py --port 8502
```

Не открывать браузер автоматически:

```bash
python3 MIRA.py --no-browser
```

## Производительность

- Стартовый экран не загружает все модели сразу.
- Загружается только выбранный режим анализа.
- Собранные модели кэшируются в памяти.
- Grad‑CAM отключён по умолчанию и включается отдельной галочкой.
- PyTorch устанавливается без CUDA на Windows и Linux, поэтому окружение меньше
  и не тратит время на поиск отсутствующей видеокарты.

Первый выбор режима может занять несколько секунд. Последующие анализы используют
кэш и выполняются заметно быстрее.

## Если не запускается

Проверьте:

1. используется 64-битный Python 3.12 или 3.13;
2. в пути проекта присутствуют `main.py` и четыре файла моделей;
3. порты `8501` и, только для команды `--certificate`, `8500` свободны;
4. телефон находится не в гостевой Wi‑Fi сети с изоляцией клиентов;
5. брандмауэр разрешает Python входящие подключения из частной сети.

Диагностика:

```bash
python3 MIRA.py --doctor
```

## Файлы проекта

- `MIRA.py` — единый установщик и launcher для Windows/Linux/macOS.
- `main.py` — интерфейс и инференс.
- `.streamlit/config.toml` — настройки интерфейса.
- `*.keras`, `*.pth` — файлы моделей.

`.venv` и `.runtime` создаются автоматически и не должны попадать в Git.

Используемые способы установки соответствуют официальным инструкциям
[TensorFlow](https://www.tensorflow.org/install/pip),
[PyTorch](https://docs.pytorch.org/get-started/locally/) и
[Streamlit](https://docs.streamlit.io/get-started/installation/command-line).
